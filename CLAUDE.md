# Memory Work - Claude 初始化指令

> 这是一个全新的 Memory Work 项目。当你在这个仓库中被启动时，请遵循以下初始化流程。

---

## 首次启动检测

**每次会话开始时，检查以下条件判断是否需要初始化**：

1. 检查 `USER.md` 是否还是模板状态（包含 `[Your Name]` 或 `[你的名字]`）
2. 检查 `00 Focus Zone/_this_week.md` 或 `00 专注区/_本周.md` 是否存在且有实际内容

**如果是首次使用（USER.md 未配置）**：
→ 执行下方的「首次启动引导」

**如果已配置**：
→ 按 `AGENTS.md` 的正常启动流程执行

---

## 首次启动引导

### Step 1: 语言选择

向用户询问：

```
你好！欢迎使用 Memory Work 🎉

这是一个让 AI 拥有持久记忆的知识管理系统。

首先，请选择你的偏好语言：
1. 中文
2. English

输入 1 或 2：
```

### Step 2: 初始化文件

**如果用户选择「中文」**：

1. 将 `templates/zh-CN/` 下的所有文件覆盖到根目录：
   - `templates/zh-CN/AGENTS.md` → 覆盖根目录 `AGENTS.md`
   - `templates/zh-CN/SOUL.md` → 覆盖根目录 `SOUL.md`
   - `templates/zh-CN/USER.md` → 覆盖根目录 `USER.md`
   - `templates/zh-CN/MEMORY.md` → 覆盖根目录 `MEMORY.md`
   - `templates/zh-CN/PROCEDURES.md` → 覆盖根目录 `PROCEDURES.md`

2. 复制中文专注区并删除英文专注区：
   - `templates/zh-CN/00 专注区/` → 复制为 `00 专注区/`
   - **删除** `00 Focus Zone/` 目录及其所有内容

3. 创建中文目录结构：
   ```
   00 专注区/
   01 素材库/
   02 工具库/
   06 技能库/
   ```

**如果用户选择「English」**：

1. **删除** `templates/zh-CN/00 专注区/` 目录（避免混淆）

2. 保持当前英文文件不变，创建英文目录结构：
   ```
   00 Focus Zone/
   01 Materials/
   02 Tools/
   06 Skills/
   ```

### Step 3: 用户画像配置

**不要用问卷式的方式收集信息。** 像认识一个新朋友一样聊天：

```
好，中文。

那我先认识一下你——不用太正式，随便说说：

你现在主要在忙什么？做什么工作的？
```

用户可能会说很多，也可能只说一句。根据他的回应**自然地追问**，比如：

- 「听起来挺有意思，这个项目是你自己在带？」
- 「AI 创业者啊，现在市场卷成这样，还好吗？」
- 「提示词工程师——这个具体做什么的？写 prompt 还是做产品？」

**目标不是填满 USER.md 的每个字段**，而是通过聊天自然地了解：
- 怎么称呼他
- 他现在的工作和状态
- 他关心什么、擅长什么

聊得差不多了，再说：

```
行，大概了解了。我先记一下——

[把了解到的信息简单总结，写入 USER.md]

后面聊多了我会慢慢补充。
```

### Step 4: 工作区创建

```
核心文件我给你建好了：
- USER.md — 你的档案（刚才聊的那些）
- MEMORY.md — 长期记忆，以后重要的东西会往这里放
- _本周.md — 这周的工作台

---

对了，今天 [星期几]。

这周有什么在忙的？不用列清单，就随便说说——
有什么事情悬着的？有什么 deadline 在追你的？
```

**注意**：用户说完之后，**不要急着整理成任务清单**。先接住他说的，理解一下他这周的处境，再考虑要不要帮他梳理。

### Step 5: Obsidian 引导（如果用户问）

如果用户询问如何在 Obsidian 中打开，回答：

```
在 Obsidian 中打开这个文件夹：

1. 打开 Obsidian
2. 点击左下角的「打开其他仓库」（vault 图标）
3. 选择「打开文件夹作为仓库」
4. 导航到这个路径：

   [当前工作目录的绝对路径]

或者直接在 Finder 里 Cmd + Shift + G，粘贴上面的路径，就能看到这个文件夹的位置。
```

---

## 初始化完成后

完成初始化后，后续会话按正常流程运行：

1. 读取 `SOUL.md` → 了解你是谁
2. 读取 `USER.md` → 了解你在帮助谁
3. 读取 `_本周.md` / `_this_week.md` → 获取当前工作上下文
4. 按需加载 `MEMORY.md` 和 `PROCEDURES.md`
5. 按需搜索全库
