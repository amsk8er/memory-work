- **执行步骤**:
  1. [步骤1]
  2. [步骤2]

- **预期结果**: [完成这个流程后的预期状态]
- **常见卡点**: [这个流程中容易卡住的地方]

**例子**：
#### 演讲课件制作流程
- **触发条件**: 需要完成一次演讲、工作坊或线上分享
- **执行步骤**:
  1. 先用 Markdown 列出核心观点和结构
  2. 将结构映射到 Reveal.js 框架
  3. 用 CSS + 简单动画做视觉层
  4. 在 1-2 次彩排时收集反馈，调整措辞和过渡
- **预期结果**: 可交付的 HTML 演讲稿，在浏览器中全屏展示
- **常见卡点**: 中间反复纠结视觉细节而忽略内容逻辑；后期时间紧张导致彩排时间不足

#### 周文件归档流程
- **触发条件**: 每周日或周一，需要整理上周的工作成果
- **执行步骤**:
  1. 从专注区扫描上周生成的新文件
  2. 对比 _本周.md 中的任务清单，标记完成和新增
  3. 将关键文件移至对应的素材库
  4. 更新 MEMORY.md 中发现的新模式
- **预期结果**: 周文件完整，专注区干净，记忆系统同步
- **常见卡点**: 时间不够，整理流程被打断；对某些文件不确定归档位置

### 沟通模式

#### [模式名称]
- **应用场景**: [什么时候用这个模式]
- **沟通要点**: [核心表达方式]
- **常见误解**: [要避免的表达方式]
- **调整历史**: [这个模式是否曾经有过调整]

**例子**：
#### 遇到想法冲突时的讨论模式
- **应用场景**: 你有新想法，但与既定方向或团队意见存在矛盾
- **沟通要点**:
  - 先阐述新想法的原因和背景
  - 再列出与既定方向的具体冲突点
  - 提出 2-3 个可能的折中方案
- **常见误解**: 不要开头就说"我有不同意见"，这会触发防守心态
- **调整历史**: 2024.10 发现对方更容易接受先理解背景再提方案的方式
